# 1 Purpose

🎯 The purpose of this document is to outline the standardized operational process for configuring and replacing camera feeds displayed on **Monitor 2, Monitor 3, and Monitor 4** through the **Smart Wall** feature of the **HikCentral Professional (HCP) Control Client**.

Unlike **Monitor 1**, which is programmed for automated camera rotation, monitors **2, 3, and 4** are primarily used for manual camera assignment using drag-and-drop placement.

This document ensures consistent procedures for live feed configuration, contributing to:

* ✅ Operational accuracy
* ✅ Improved situational awareness
* ✅ Reliable surveillance coverage in the Security Operations Room

---

# 2 Scope

📌 This document applies to:

* 👨‍💻 IT Department personnel authorized to modify Smart Wall configurations
* 🛡️ Security Operations Room staff responsible for monitoring camera feeds
* ⚙️ System administrators overseeing surveillance infrastructure

> ❗ The scope does not cover automated rotation configuration (**Monitor 1**) or video recorder administration.

---

# 3 System Description

🏢 The Security Operations Room operates an integrated **HikCentral Smart Wall monitoring environment**, consisting of four physical display monitors arranged in a **2×2 grid**.

## 📺 Monitor Layout Overview

| Physical Monitor | Display Position | Smart Wall Identifier | Configuration Method |
| ---------------- | ---------------- | --------------------- | -------------------- |
| Monitor 1        | Upper-left       | 100                   | Auto-switch mode     |
| Monitor 2        | Upper-right      | 200                   | Manual drag-and-drop |
| Monitor 3        | Lower-left       | 300                   | Manual drag-and-drop |
| Monitor 4        | Lower-right      | 400                   | Manual drag-and-drop |

📌 This guide specifically covers the configuration process for **Monitors 2, 3, and 4**.

# 6 Activating Live Camera Preview

⚠️ Camera feeds may not appear until this option is manually enabled.

## ▶️ Procedure

### a.

Navigate back to the **Smart Wall Dashboard**.

---

### b.

Locate the toggle:

```text
Display Camera Image on Client
```

(Location: Bottom-right of interface)

---

### c.

Click the radio button to activate:

* ⚪ Grey = Disabled
* 🟢 Green = Enabled (live preview active)

---

### d.

Once activated, all configured monitors will begin displaying live surveillance imagery.

> 📌 Note:
> This function must remain active to visually verify camera alignment and operational status.

---

# 7 Replacing the camera from the monitor 3

> 📌 “The same process applies to Monitors 2 and 3.”

This example demonstrates replacing a single camera feed in **Monitor 4**, position **401**.

---

# 7.1 Step-by-Step Procedure

## i. Ensure Network Source Mode

Ensure you are in **Network Source Mode**.

---

## ii. Locate Camera Search Field

In the dashboard, locate the camera search field.

---

## iii. Identify Existing Camera

Identify the existing camera assigned to position **401**.

### • For Example:

```text
Office_Entry_Gate
```

> 📌 Note:
> This will show by hovering in the camera.

---

## iv. Search Replacement Camera

Enter the replacement camera name into the search bar.

### • Example:

```text
Office_Cafeteria
```

---

## v. Replace Camera Feed

From the search results, drag and drop the new camera onto position **401**.

---

## vi. Verify Replacement

When hovering over slot **401**, the placeholder will update to show the new camera name.

---

# 8 Saving Configuration

## 💾 Saving Procedure

### a)

Click the **Save** icon in the lower navigation bar.

---

### b)

Select **Save** (not Save As) to overwrite the existing configuration.

---

### c)

A confirmation message:

```text
Saving Succeeded
```

will appear.

---

# 9 Verification

## ✅ Post-Configuration Verification

Once saved:

* ✅ The new camera feed will display live on **Monitor 4, position 401**
* ✅ The prior camera feed is permanently replaced until reconfigured

> 💡 Tip:
> Use consistent naming identifiers when searching for cameras to reduce configuration errors.
