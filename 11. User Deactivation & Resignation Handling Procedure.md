# 1. 🧾 Introduction

This document defines the standardized procedure for **user deactivation and resignation handling** within the HCP system (Hik – Central Professional).

It ensures that employees who have resigned or completed their tenure are **systematically removed from physical access control**, including:

* 🧠 Fingerprint authentication
* 💳 Card-based access

Once the process is completed, user access remains valid only until **23:59:00 of the current working day**, after which all physical access is automatically revoked.

---

# 2. 🎯 Purpose

This SOP is designed to:

* 🔐 Strengthen physical security across office premises
* ⏱ Ensure timely removal of access for resigned employees
* 📋 Standardize IT access revocation procedures
* 🛡 Maintain compliance with internal security policies

---

# 3. 📌 Scope

This procedure applies to:

* 👨‍💻 IT Department (INFRA & SECURITY Team)
* 🏢 HCP Physical Access Control System
* 👥 All resigned, terminated, or end-of-contract employees

# 6. ⚙️ Standard Operating Procedure (SOP)

---

## 6.1 🔑 Login to HCP System

1. Open a web browser 🌐
2. Navigate to: [https://172.16.1.1/#/](https://172.16.1.1/#/)
3. Enter credentials:

   * Username: admin
   * Password: **********
4. Click **Login** to access the dashboard

---

## 6.2 🔍 Search for User

1. From dashboard, click **Person**
2. User management dashboard will be displayed
3. Locate and use the **Filter/Search option**
4. In the filter field, enter: **Resigned**
5. Click **Filter** 🔎
6. Matching user results will be displayed

📌 *Note: Ensure correct spelling and ID format during search.*

---

## 6.3 👁 View User Details

1. From search results, click on **User ID**
2. The full user profile dashboard will open
3. Verify user identity and employment status

---

## 6.4 📅 Modify Effective Period

1. Locate **Effective Period** section
2. Click on 📆 calendar icon
3. Set **End Date = Today’s Date**
4. Click **OK** to confirm

📌 This ensures access expires at end of the current day.

---

## 6.5 💾 Save Configuration

1. Click on the **Save (💾)** icon
2. System will redirect back to dashboard

✔️ **Status:** Configuration successfully saved

---

## 6.6 🚫 Move User to Resigned Section

1. Navigate to **Resigned Section** from left menu

2. Click **➕ Add**

3. A pop-up panel will appear

4. Click on **User Add Icon** 👤

5. Search user by ID (e.g., 9999999999999999)

6. Select user via checkbox ☑️

7. Click **ADD**

8. Select **Resignation Type**:

   * ❌ Dismiss
   * 📝 Resignation
   * 🔄 Redeployment
   * ⛔ Suspension with Pay

9. Enter departure reason (if applicable)

10. Click **OK** to finalize

---

## 6.7 ✔️ Verification

* Confirm user appears in **Resigned List**
* Ensure effective date is updated correctly
* Validate system status synchronization

---

## 6.8 🚪 Logout Procedure

1. Click on username (top-right corner)
2. Select from menu:

   * 🔑 Change Password
   * 🚪 Logout
3. Click **Logout**
4. Confirm by clicking **OK**

# 7. 📊 Expected Outcome

After successful completion:

* ⏳ User access remains valid until **23:59:00 (same day)**
* 🧠 Fingerprint access is automatically disabled
* 💳 Card access is automatically disabled
* 🚷 User cannot access office premises thereafter
