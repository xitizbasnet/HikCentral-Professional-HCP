# 📖 Overview

This guide explains how to assign an **Attendance Access Level** to a user in the HCP Access Control System.

The procedure includes:

* Logging into the HCP portal
* Locating a user account
* Editing access level permissions
* Assigning attendance access
* Verifying attendance device access

# 🚀 Procedure: Add Attendance Access Level

---

# Step 1 — Login to HCP

1. Open a web browser.
2. Navigate to:

```text id="f8r4qe"
https://172.16.1.1/#/
```

3. Enter the administrator credentials.
4. Click:

```text id="m1g0yb"
Login
```

---

# Step 2 — Open Person Management Section

After successful login:

1. Navigate to:

```text id="x2g8uw"
Person
```

---

# Step 3 — Search and Select User

1. The list of users/persons will appear in the middle search result section.
2. Locate the required user.

### Example User

| Field       | Value          |
| ----------- | -------------- |
| **Name**    | `Xitiz Basnet` |
| **User ID** | `703`          |

3. Click on the user name to open the edit configuration page.

---

# Step 4 — Open Access Level Settings

1. Navigate to the:

```text id="r9k7vl"
Access Level
```

section.

2. Scroll down until you locate the option:

```text id="b5w2ex"
+Assign
```

under the Access Level configuration area.

---

# Step 5 — Assign Attendance Access Level

1. Click:

```text id="v6j1md"
+Assign
```

2. A new dashboard/window will appear displaying the list of available access levels configured in the system.

3. From the available options:

* Locate the access level named:

```text id="o8h4qy"
Attendance
```

4. Tick/check the checkbox next to the Attendance access level.

---

# Step 6 — Confirm Assignment

1. Click:

```text id="z1t6po"
Assign
```

2. The system should now display the Attendance access level assigned to the selected user.

### Example

```text id="v4s9re"
Attendance access level successfully added to user: Xitiz Basnet
```

---

# Step 7 — Save Configuration

1. Click:

```text id="j0m3da"
Save
```

to apply the changes.

---

# ✅ Verification

After saving the configuration:

1. Test the user’s card/device access on the attendance device.
2. Verify that:

   * The attendance device recognizes the card
   * Access is granted successfully
   * Attendance logs are recorded properly

# 🔍 Troubleshooting Tips

## Attendance Device Does Not Accept Card

Verify the following:

* Access level was properly assigned
* User card is active
* Attendance device is online
* Synchronization between HCP and attendance device is successful

---

## Access Level Not Visible

Possible causes:

* Missing administrator permissions
* Access level not configured in system
* Database synchronization issue

---

## Save Button Not Working

Check for:

* Required fields missing
* Browser session timeout
* Network connectivity issues
