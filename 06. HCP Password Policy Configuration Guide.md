# 📖 Introduction

This guide provides step-by-step instructions for configuring and reviewing the password policy in the HCP (HIK Central Professional) login portal.

It ensures that password security settings align with organizational security standards, including complexity rules, expiration policies, and password strength enforcement.

---

# ⚠️ Security Notice

> 🔒 Important
> Password policy settings directly impact system security. Ensure all changes comply with your organization’s security guidelines before applying updates.

---

# 🚀 Step-by-Step Configuration Guide

---

# Step 1 — Access the HCP Login Portal

1. Open your web browser.
2. Navigate to the HCP portal:

```text id="k2v8xp"
https://172.16.1.1/#/
```

---

# Step 2 — Log in with User Credentials

1. Enter your login credentials:

| Field    | Description       |
| -------- | ----------------- |
| Username | Your HCP username |
| Password | Your HCP password |

2. Click:

```text id="a7m4qz"
Login
```

to access the system.

---

# Step 3 — Access Account & Security Settings

1. After login, locate your:

```text id="v3n9lx"
Account / Profile
```

section.

2. Open the dropdown or menu panel.
3. Select:

```text id="p8r2td"
Account and Security
```

---

# Step 4 — Navigate to Security Settings

1. On the left-hand navigation panel, locate:

```text id="s1k6wf"
Security Settings
```

2. Click:

```text id="d9x5qm"
Basic Parameters
```

to open password policy settings.

---

# Step 5 — Review and Update Password Policy

Review the current configuration under **Basic Parameters**.

---

## 🔐 Password Policy Settings May Include:

* Minimum password length
* Password complexity rules:

  * Uppercase letters
  * Lowercase letters
  * Numbers
  * Special characters
* Password expiration period
* Account lockout settings
* Other security parameters

---

## ✏️ Update Settings

1. Modify the required fields based on organizational security requirements.
2. Ensure all changes follow internal IT security standards.

---

# Step 6 — Save and Verify Changes

1. Click:

```text id="z6t1cn"
Save
```

to apply changes.

2. Log out of the HCP portal.
3. Log back in to confirm:

* New password policy is active
* Settings are correctly applied
* No configuration errors exist

---

# 📌 Important Notes

* Always follow organizational password security policies.
* Do not reduce password complexity below approved security standards.
* Any misconfiguration may affect user authentication.
* Contact the system administrator if issues occur.

# 🛠️ Troubleshooting

## Unable to Save Settings

Possible causes:

* Insufficient admin privileges
* Browser session timeout
* Missing required fields

---

## Policy Not Applied

Check:

* Whether changes were saved correctly
* If system cache needs refresh
* If login session was restarted

---

## Settings Not Visible

Ensure:

* Correct admin account is used
* Security module access is enabled
* Role permissions are assigned
