# 📖 Objective

This document provides a standardized procedure to convert:

> **Windows Server 2016 Evaluation Edition → Windows Server 2016 Standard Edition**

The process includes:

* Edition conversion using DISM
* KMS key installation
* Server activation
* Verification of licensing status

---

# ⚠️ Prerequisites

Before proceeding, ensure the following requirements are met:

| Requirement      | Description                            |
| ---------------- | -------------------------------------- |
| Operating System | Windows Server 2016 Evaluation Edition |
| Permissions      | Administrator access required          |
| Product Key      | Windows Server 2016 Standard KMS Key   |
| Network Access   | Required for KMS activation            |

# 🚀 Conversion Procedure

---

# Step 1 — Open Command Prompt as Administrator

1. Click **Start Menu**.
2. Search:

```text id="c8m4xz"
cmd
```

3. Right-click **Command Prompt**.
4. Select:

```text id="v1q7nt"
Run as administrator
```

---

# Step 2 — Convert Edition to Windows Server 2016 Standard

Run the following command in elevated Command Prompt:

```cmd id="d5r8kp"
dism /online /set-edition:ServerStandard /productkey:WC2BQ-8NRM3-FDDYY-2BFGV-KHKQY /accepteula
```

---

## 🔄 Expected Behavior

* System initiates edition upgrade
* Accepts license terms automatically
* Prompts for restart after completion

---

## 🔁 Reboot Requirement

When prompted:

```text id="r9k2lm"
Press Y
```

Then:

* Restart the server **twice**
* Ensure full boot completion after each restart

---

# Step 3 — Verify Edition Upgrade

After reboot:

## Check Windows Version

Run:

```cmd id="x7t1qa"
winver
```

### Expected Result:

* Windows Server 2016 Standard Edition should be displayed

---

# Step 4 — Install KMS Product Key

Run the following command:

```cmd id="n2v6dp"
slmgr /ipk WC2BQ-8NRM3-FDDYY-2BFGV-KHKQY
```

---

# Step 5 — Configure KMS Server

Set the KMS server address:

```cmd id="m6q8wx"
slmgr /skms kms8.msguides.com
```

---

# Step 6 — Activate Windows Server

Execute activation command:

```cmd id="t3p9zr"
slmgr /ato
```

---

# Step 7 — Confirm Activation Status

Verify activation using:

```cmd id="y8k1nb"
slmgr /dli
```

---

## ✅ Expected Output

* License status: **Licensed**
* KMS activation confirmed

---

# 📦 Final Command Summary

Run in order:

```cmd id="f4x7lm"
dism /online /set-edition:ServerStandard /productkey:WC2BQ-8NRM3-FDDYY-2BFGV-KHKQY /accepteula
```

# ⚠️ Important Notes

* Ensure all critical applications are functioning after conversion
* Verify server roles and services post-upgrade
* Always maintain a backup before system-level changes
* Ensure network connectivity during activation

---

# 🛠️ Troubleshooting

## Activation Failure

Check:

* KMS server connectivity
* Correct product key usage
* DNS resolution

---

## Edition Not Updating

Possible causes:

* Pending reboot not completed
* Incorrect DISM syntax
* Insufficient admin privileges

---

## Winver Not Reflecting Changes

* Restart system again
* Allow full boot cycle completion
