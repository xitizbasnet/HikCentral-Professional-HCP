# 👤 Procedure: Enroll New Employee in HCP Server

---

# Step 1 — Login to HCP

1. Open a web browser.
2. Navigate to:

```text id="hcp_login_01"
https://172.16.1.1/#/
```

3. Enter administrator credentials.
4. Click **Login**.

---

# Step 2 — Open Person Management Module

After successful login:

1. Navigate to:

```text id="person_module_01"
Person
```

---

# Step 3 — Add New User

1. Click:

```text id="add_user_01"
+ Add
```

2. Open the user creation form.

---

# Step 4 — Enter Employee Details

Fill in the required employee information.

---

## 🧾 User Information Fields

| Field                  | Description                                          |
| ---------------------- | ---------------------------------------------------- |
| **ID Number**          | Unique employee ID (must not duplicate existing IDs) |
| **Department**         | Employee department                                  |
| **First Name**         | Employee first name                                  |
| **Last Name**          | Employee last name                                   |
| **Effective Date**     | Start date of system access                          |
| **Date of Employment** | Official joining date                                |

---

## 🧑 Example Entry

| Field              | Value        |
| ------------------ | ------------ |
| ID Number          | `123`        |
| Department         | IT           |
| First Name         | Xitiz        |
| Last Name          | Basnet       |
| Effective Date     | (Enter date) |
| Date of Employment | (Enter date) |

> ⚠️ Important
> Each user ID must be **unique** and cannot be duplicated in the system.

---

# Step 5 — Save User Profile

1. After entering all required details, click:

```text id="save_user_01"
Save
```

2. The system will register the new employee in HCP.

---

# Step 6 — Verify User Creation

To confirm successful enrollment:

1. Search for the newly created user in the **Person** list.
2. Verify:

   * ID is correctly assigned
   * Name is correctly displayed
   * Department is correct
   * User is active

---

# 📌 Expected Result

After successful completion:

* New employee is registered in HCP
* Unique ID is assigned
* User appears in access control list
* Employee is ready for access level assignment (if required)

# 🛠️ Troubleshooting

## User Not Saving

Possible causes:

* Missing required fields
* Duplicate ID number
* Session timeout

---

## User Not Visible After Creation

Check:

* Search filters
* Page refresh
* Proper save confirmation

---

## Duplicate ID Error

* Ensure the ID is unique before saving
* Cross-check existing user database

---

# 🔒 Security Recommendations

* Restrict admin access to authorized IT staff only
* Ensure proper validation before saving user records
* Avoid reusing employee IDs
* Maintain audit logs of all user creation activity

---

# 🏁 Conclusion

This procedure ensures proper onboarding of new employees into the HCP Access Control System.

After completion, the new employee:

* Is registered in the system
* Has a unique ID assigned
* Can be configured for access permissions and attendance access
